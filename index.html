Deterministic Git cleanup and release checkout

Protect your release process with a clean, reproducible local state.

Release Clean is a focused Python CLI that resets tracked changes, removes ignored files based on .gitignore, preserves node_modules/, synchronizes main, and positions your repository on the correct release/<VERSION> branch with a predictable sequence.

Why Release Clean exists

Local environments accumulate hidden state. Ignored files, leftover artifacts, outdated branches, and tracked changes can all distort release validation. Release Clean replaces manual cleanup with a deterministic, repeatable workflow that reduces uncertainty before you operate on a release branch.

Deterministic by design

Uses Git-native commands to clean ignored files, reset tracked changes, and synchronize the repository before release checkout.

Controlled execution

Shows a clear pre-execution summary, asks for explicit confirmation, and stops immediately on the first failure.

Practical for real teams

Preserves node_modules/ while still removing ignored files that commonly pollute release validation workflows.

Executed workflow

The sequence is intentional. It cleans ignored files, resets tracked state, synchronizes the base branch, checks out the target release branch, resets again, and updates it from origin. 

git clean -fdX -e node_modules/
git checkout -- .
git checkout main
git pull
git fetch --all
git checkout release/<VERSION>
git checkout -- .
git pull origin release/<VERSION>

Key features

Repository-aware cleanup

Removes ignored files based on .gitignore using git clean -fdX, while excluding node_modules/.

Tracked reset

Runs git checkout -- . before and after release checkout to prevent local state from leaking into validation.

Validation first

Checks the current directory is inside a Git repository and validates the version format before building the branch name.

Audit-friendly output

Records executed commands and prints a final summary with timestamps, repository path, version, branch, and status counts.

Workflow rationale

Clean ignored files
Removes local artifacts defined by the repository itself, not by hardcoded assumptions.
Reset tracked changes
Ensures the working tree returns to a known state before branch synchronization.
Sync main
Updates the base branch so release checkout begins from an aligned local context.
Reinforce state after release checkout
A second reset helps guarantee the release branch is not affected by leftover local state.

Install

The recommended way to install Release Clean as a global CLI is with pipx.

python3 -m pip install --user pipx
python3 -m pipx ensurepath

# restart your terminal
pipx install release-clean

Local development

To test from the project directory during development:

pip3 install -e .
release-clean

Usage

release-clean

Run the command from anywhere inside a Git repository. The utility will prompt for the version and then display the exact sequence it plans to execute.
